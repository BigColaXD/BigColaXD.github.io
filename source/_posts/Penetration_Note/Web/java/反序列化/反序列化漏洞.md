# 1.什么是java反序列化漏洞？

Java反序列化漏洞是指攻击者通过操纵序列化数据，在反序列化过程中执行恶意代码的安全漏洞。

## 序列化与反序列化

序列化：将Java对象转换为字节流，便于存储或传输。

反序列化：将字节流恢复为Java对象。

## 漏洞成因

不受控的反序列化：当应用程序反序列化不可信数据时，攻击者可构造恶意字节流，导致任意代码执行。

常见问题类：某些类（如ObjectInputStream）在反序列化时会自动执行代码，攻击者可利用这些类执行恶意操作。

## 攻击方式

构造恶意序列化数据：攻击者生成包含恶意代码的序列化数据。

触发反序列化：通过输入或网络请求将恶意数据传递给应用程序，触发反序列化过程。

## 影响

远程代码执行：攻击者可在目标系统上执行任意代码。

数据泄露：可能导致敏感信息泄露。

系统控制：攻击者可能完全控制系统。

## 防御措施

避免反序列化不可信数据：确保反序列化的数据来源可信。

使用白名单：限制可反序列化的类。

安全库：使用如SerialKiller等库过滤恶意数据。

更新依赖：及时修复已知漏洞的库和框架。

## 示列

```java
public class VulnerableClass implements Serializable {
    private String command;

    private void readObject(ObjectInputStream in) throws IOException, ClassNotFoundException {
        in.defaultReadObject();
        Runtime.getRuntime().exec(this.command);
    }
}
```

攻击者可构造恶意序列化数据，在反序列化时执行任意命令。**注意攻击者在攻击过程中只能控制command成员变量。VulnerableClass类要在本地环境存在才能命令执行漏洞利用成功。**
