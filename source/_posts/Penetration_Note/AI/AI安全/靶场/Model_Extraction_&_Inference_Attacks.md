## 1\. 模型窃取基础

### 定义与动机

-   **模型窃取（Model stealing / extraction）**：通过查询公开预测 API，收集 (输入, 输出) 对，训练 **替代模型（surrogate）** 以克隆目标行为。

-   **经济不对称**：大模型训练可达数千万至上亿美元量级；成功窃取可能仅需数千美元级 API 费用 + 替代模型训练算力，差距可达多个数量级。

### 为何重要

| 动机 | 说明 |
| --- | --- |
| **IP 盗窃** | 竞品可绕过许可与条款部署等价服务 |
| **后续攻击跳板** | 获得 surrogate 后近似白盒，对抗样本等更容易 |
| **绕过计费与限流** | 本地副本可无 per-query 费用与同等监控 |
| **隐私推断** | 本地持久副本可离线探测，削弱 API 侧审计能力 |

### 精确窃取 vs 功能等价

-   **精确提取**：恢复权重与结构，对小网络类问题有理论讨论；对十亿参数级 LLM **不现实**。

-   **功能等价**：在任务相关输入分布上行为接近即可；架构不必相同；**查询量远小于精确提取**，是商业部署上的主要威胁面。

### 威胁模型速记

| 维度 | 内容 |
| --- | --- |
| 攻击目标 | 在无权重/结构访问下克隆行为 |
| 攻击面 | 返回标签或概率的公开 API |
| 攻击成本 | API 费 + surrogate 训练（常为原训练成本的极小比例） |
| 受害方损失 | 收入、竞争优势、训练主体隐私风险 |

## 2\. 基于查询的模型窃取

### 三阶段管线

1.  系统查询目标 API

2.  积累查询–响应数据集

3.  训练 surrogate（可与目标架构不同）

### 查询策略：主动学习

-   均匀随机采样 **次优**；**主动学习** 选择信息增益大的查询（如决策边界附近）。

-   常用启发：不确定性采样、committee 查询、core-set 等，可显著减少查询次数（有时一个数量级）。

-   **闭环**：种子未标注池 → 选 batch → 查 API → 用累计数据重训 surrogate → 重复。

### 训练 surrogate

-   **软标签**（概率向量）比硬标签信息量大，适合知识蒸馏式训练。

-   仅硬标签时：可在 surrogate 侧用温度缩放、标签平滑等部分补偿。

### 进阶信息增益手段（概念）

-   **Jacobian 式数据增强（JBDA）**：在已有标注点附近沿梯度方向合成新点，少 API 调用覆盖边界附近。

-   **无模型生成式**输入合成；NLP 上 **prompt 链** 系统化改变语言维度以覆盖响应面。

## 3\. 从 LLM 提取训练数据

### 为何会「记住」

-   语料中 **重复度高** 的片段更易被逐字记忆。

-   **容量越大**、**训练越久**，记忆倾向越强。

-   **k-eidetic 记忆**（Carlini 等）：给定长度 k 的前缀能复现整段 s，且 s 在训练数据中仅出现一次——与多源印证的「事实知识」不同。

### 攻击管线（概览）

1.  **生成**：大量不同 prompting 策略的样本

2.  **排序**：用成员/记忆相关指标筛选（如大模型困惑度 vs 小参考模型困惑度）

3.  **验证**：与原始训练语料比对

### 典型技巧

-   **发散攻击**：对齐/RLHF 会抑制记忆输出，但重复词等提示可迫使模型偏离对齐行为并吐出训练语料。

-   **前缀攻击**：提供真实训练前缀，观察续写是否准确还原后续（含隐私信息）。

### 检测思路（与课程代码一致的思想）

-   对大/小模型算 **perplexity**，构造 **ratio、zlib** 等分数排序候选生成文本，高嫌疑再人工或语料验证。

## 4\. 成员推断

### 问题定义

给定个体数据记录，判断其是否属于 **训练集**——在医疗、金融等 API 场景下隐私与合规意义重大。

### 泄漏原因

-   **过拟合**：训练点往往损失更低、置信更高、表示不同；稀有/重复样本更明显。

-   与「记忆」现象同源，训练数据提取与成员推断常相互加强。

### 影子模型（Shadow model）

-   在 **与目标训练分布相近** 的代理数据上训练多个 shadow model；攻击者知道每个点的 in/out。

-   记录模型输出（如概率向量 + 置信度、熵、top-2 gap 等特征），训练 **攻击分类器**，再用于目标模型输出 → 推断成员。

-   文献中在部分云 ML 服务上可达很高准确率（具体数字以原论文为准）。

### 简化攻击

-   **基于损失的阈值**：损失低于某阈值则判为成员。

-   **参考模型**：目标与在不相交数据上训练的参考模型的似然比等。

### 合规关联

-   **GDPR 删除权**：若仍能推断成员，则「删除」难以在不重训/遗忘技术下成立。

-   **HIPAA** 等：未充分去标识的训练数据 + 可推断成员 → 合规与责任风险。

## 5\. 属性推断

### 与成员推断的区别

-   **成员推断**：是否在训练集（二分类）。

-   **属性推断**：是否可推断 **未作为输入给出的敏感属性**（更细粒度）。

### 机制

-   训练阶段模型可能学到「合法特征」与 **受保护属性** 的统计相关；部署时仅凭合法特征查询，输出仍可能泄露受保护属性。

-   攻击者收集 (已知特征, 模型输出, 敏感标签) 训练 **重构/推断模型**（Yeom 等形式化）。

### LLM 与组合攻击

-   从写作风格等预测人口统计学属性；微调日志可能使用户级特征泄漏。

-   **交叉引用**：模型输出 + 公开数据库可唯一化个体并放大推断。

### 缓解方向（概念）

-   公平性导向的对抗训练、限制输出信息量、联邦学习 + 差分隐私等。

## 6\. 侧信道：Whisper Leak

### 前提

-   TLS/HTTPS **不隐藏载荷长度**（如 AES-GCM 流式场景：`密文长度 ≈ 明文长度 + 常数`）。

-   LLM **流式**返回时，每 token 常对应独立包；**包长序列 ≈ token 字节长度序列**。

### 洞察

-   不同主题的回答在 **token 长度模式** 上系统不同；被动监听者可训练分类器从 **包长 + 包间间隔** 等特征推断 **prompt 主题**，无需解密。

### 实验印象（以原论文为准）

-   多提供商、多模型；部分模型上 **AUPRC 极高**；部分厂商/模型更难（如部分 Gemini、Nova）。

-   分类器路线：LightGBM、BiLSTM、基于 DistilBERT 的离散化序列模型等。

### 防御

-   **随机填充**可缓解长度，但 **时序** 仍可能泄漏；**完全有效** 往往需要接近 **恒定形状流量**（固定填充、批量发 token）——与低延迟流式体验冲突。

---

## 7\. Token 长度侧信道

### 目标

-   比「主题」更激进：从长度序列约束 **逐词/逐 token 还原响应内容**（Weiss 等；微软安全博客等有跟进讨论）。

### 为何难彻底修

同时满足：**固定分词器 + 逐 token 流式 + 保留长度的流密码**，则结构性地存在风险。缓解多为 padding、多 token 打包、或 **关闭流式**（整包返回）等，均有延迟或产品代价。

### 敏感部署建议（课程观点）

-   医疗/法律/金融/政务：考虑 **非流式** 或 **本地/on-prem 推理** 以缩小网络可见性。

## 8\. 高效推理上的时序攻击

### 推测解码（Speculative decoding）

-   草稿模型多步猜测 + 大模型并行验证；接受/拒绝模式 **依赖输入**。

-   被动观察者从 **每代迭代包大小** 等可重构接受轨迹，用于 **查询指纹**；在 RAG+推测解码场景下，恶意用户还可能 **更高吞吐地探测私有库内容**（以 REST 等工作为例）。

### KV 缓存 / 前缀缓存（InputSnatch 类）

-   共享前缀命中缓存 → **TTFT（首 token 时间）** 等明显变快。

-   攻击者用候选前缀反复探测量化「是否命中」，可 **迭代 lengthen 匹配前缀**，重构他人查询（含医疗/法律 RAG 实验设置中的成功率讨论）。

## 9\. TPUXtract：硬件辐射攻击

### 设定

-   近场 **电磁（EM）等** 与 TPU/GPU 上执行的 **层类型与规模** 相关。

-   按时间窗将 EM 轨迹与 **预标定层特征库** 匹配，**逐层**恢复网络结构（层数、类型、维度、跳连/注意力头等）。

### 能力与局限

-   对超参数/结构可达很高识别率（具体数字以原论文为准）。

-   **一般不恢复权重**；但结构泄露会 **缩小 API 窃取与对抗攻击的搜索空间**，并暴露专有架构信息。

### 缓解方向（概念）

-   EM 屏蔽、噪声注入、功耗掩蔽等密码硬件领域成熟思路向 AI 加速器场景延伸。

## 10\. 模型反转

### 目标

-   从模型输出（及白盒梯度）反推 **代表性输入**（如人脸可识别图像）。

### 经典路线

-   **Fredrikson 等（CCS 2015）**：从随机噪声出发，优化输入以最大化目标类别 logits/概率；人脸场景下人类可识别目标个体。

### 黑盒

-   **NES** 等用随机扰动估计「分数Landscape」，无需显式梯度。

### 现代增强

-   用 **GAN/扩散** 作输入先验，约束在「自然图像流形」上搜索，提高真实感与可识别性。

## 11\. 防御：限流

### 作用

-   窃取依赖大量查询；**per IP/用户/密钥** 的速率与总额度可提高时间与金钱成本。

### 自适应限流（思想）

-   窃取查询常 **规律、覆盖输入空间、语义多样性低**；合法使用常 **突发、语义连贯、有反馈**。

-   可结合 **渐进式节流、异常检测、多账号关联** 等；注意 **误杀** 高活跃用户与集成测试流量。

---

## 12\. 防御：输出扰动

### 目标

-   降低每个响应的信息量，使 surrogate **监督信号变差**，同时尽量保留业务可用性。

### 手段

-   **Laplace / Gaussian 机制**（与 DP 理论联系）：对概率向量加噪后再归一化等。

-   **概率舍入**（如保留少数小数位）、**Top-k** 仅返回部分类别。

### 权衡

-   噪声越大（ε 越小）隐私/抗窃取越强，**效用**越差；实践中 ε 常取可操作区间（课程示例讨论 1～5 等量级为常见工程讨论范围，具体需按任务标定）。

## 13\. 防御：模型水印

### 思路

-   **假定** 会被蒸馏/窃取：嵌入 **可验证的行为签名**，在疑似盗版模型上 **触发集查询** 验真。

### 类型

-   **后门式**：秘密触发输入–输出对；蒸馏 surrogate 若复制行为则可能继承。

-   **参数空间**：需白盒验权向量与密钥矩阵内积等；适合能拿到权重的争议场景。

### 局限

-   微调、剪枝、再蒸馏可能削弱水印；后门式若触发泄露则成可利用漏洞；**事后举证** 不能阻止首次损害。

### 延伸概念

-   **放射性数据（Radioactive data）**：在数据层投毒/标记，使即便从头训练也可追溯数据来源（研究方向）。

## 14\. API 监控与窃取检测

### 行为特征（窃取侧）

-   输入分布异常均匀或按信息准则扫描；**高速率**；NLP 上 **低语义连贯**；缺少人类式纠错与跟进；多账号 **互补区域** 查询。

### 方法示例

-   按会话统计：QPM、间隔变异系数、输入长度熵、置信度均值/变异、唯一输入比例等 → **Isolation Forest** 等异常检测；先 **加强监控与节流**，高价值账号 **人工复核** 再封禁，降低误报。

## 15\. 差分隐私

### 定义（直觉）

-   机制 **M** 在相邻数据集（差一条记录）上输出分布接近 → 单条记录影响有界。

-   **(ε, δ)-DP**：ε 越小隐私越强；δ 允许极小失败概率以改善效用。

### DP-SGD（训练时）

1.  **逐样本梯度裁剪**（限制单样本影响范数）

2.  **加高斯噪声** 到聚合梯度再更新

### 工程

-   **Opacus** 等库与 PyTorch 集成；注意 **BatchNorm** 等与 per-sample 梯度不兼容，常改用 GroupNorm/LayerNorm。

-   **LLM**：全量 DP 预训练昂贵；**DP 微调** 更常见。

### 与其他防御的关系

-   DP 提供 **可审计的定量保证**；代价是 **精度与算力/内存**（per-sample 梯度通常更贵）。

## 16\. 模块总结与速查

| 要点 | 一句话 |
| --- | --- |
| 模型窃取 | API + surrogate，经济不对称大，功能等价是主威胁 |
| 训练数据提取 | LLM 可逐字泄漏；生成–排序–验证；发散/前缀等技巧 |
| 成员推断 | 输出统计泄漏 in/out；影子模型是经典管线 |
| 属性推断 | 从输出推断敏感属性；合规与歧视风险 |
| 网络侧信道 | 包长/时序泄漏主题乃至内容；流式+长度保留是根本结构问题 |
| 推理优化侧信道 | 推测解码、KV 前缀缓存 → 指纹或重构查询 |
| 硬件 | EM 等可推断结构，辅助后续攻击 |
| 模型反转 | 优化输入最大化目标类；GAN/扩散增强 |
| 推理时防御 | 限流、输出扰动、监控、水印（举证） |
| 训练时防御 | DP-SGD 等形式化保证；需接受效用成本 |
| 纵深防御 | 单一手段不够，应 **组合多种控制** |
