# 总结

| **Port Scan Type** | **Example Command** |
| --- | --- |
| TCP Connect Scan | nmap -sT 10.10.152.123 |
| TCP SYN Scan | sudo nmap -sS 10.10.152.123 |
| UDP Scan | sudo nmap -sU 10.10.152.123 |

| **Option** | **Purpose** |
| --- | --- |
| -p- | 所有端口 |
| -p1-1023 | 扫描端口 1 至 1023 |
| -F | 100 个最常见的端口 |
| -r | 按连续顺序扫描端口 |
| -T<0-5> | -T0 最慢，T5 最快 |
| --max-rate 50 | 速率 <= 50 个数据包/秒 |
| --min-rate 15 | 速率 >= 15 个数据包/秒 |
| --min-parallelism 100 | 至少 100 个探头并行 |

Nmap 考虑以下六种状态：

1.  **Open**: 服务正在侦听指定的端口。
2.  **Closed**: 没有服务正在侦听指定的端口，尽管该端口是可访问的。通过可访问，我们的意思是它是可访问的，并且不会被防火墙或其他安全设备/程序阻止。
3.  **Filtered**: 无法确定端口是打开还是关闭，因为端口不可访问。此状态通常是由于防火墙阻止 Nmap 访问该端口。Nmap 的数据包可能会被阻止到达端口;或者，响应被阻止到达 Nmap 的主机。
4.  **Unfiltered**: 无法确定端口是打开还是关闭，尽管该端口是可访问的。  
    使用 ACK 扫描 `-sA` 时会遇到此状态。
5.  **Open|Filtered**: 无法确定端口是打开还是已过滤。
6.  **Closed|Filtered**: 这意味着 Nmap 无法决定端口是关闭还是过滤。

DNS 服务默认使用 UDP 端口 53

SSH 服务默认使用 TCP 端口 22

# TCP Flags

TCP 标头是 TCP 段的前 24 个字节。下图显示了 RFC 793 中定义的 TCP 标头。

在第一行中，我们有源 TCP 端口号和目标端口号。我们可以看到端口号被分配了 16 位（2 字节）。

在第二行和第三行中，我们有序列号和确认号。每行分配了 32 位（4 个字节），总共 6 行，共 24 个字节。

![](https://cdn.nlark.com/yuque/0/2024/png/39170111/1716554164626-69f9b89f-e809-4212-8c74-c63b46baddc3.png)

特别关注 Nmap 可以设置或取消设置的标志。我们以红色突出显示了 TCP 标志。设置标志位意味着将其值设置为 1。

从左到右，TCP 标头标志为：

1.  **URG**: 紧急标志表示紧急指针归档很重要。紧急指针指示传入数据是紧急的，并且会立即处理设置了 URG 标志的 TCP 段，而无需考虑等待以前发送的 TCP 段。
2.  **ACK**: 确认标志表示确认编号很重要。它用于确认收到 TCP 段。
3.  **PSH**: 推送标志，要求 TCP 及时将数据传递给应用程序。
4.  **RST**: 重置标志用于重置连接。其他设备（如防火墙）可能会发送它来断开 TCP 连接。当数据发送到主机并且接收端没有要应答的服务时，也会使用此标志。
5.  **SYN**: 同步标志用于启动 TCP 3 向握手，并与其他主机同步序列号。在建立 TCP 连接期间，应随机设置序列号。
6.  **FIN**: 发送方没有更多数据要发送。

# TCP Connect Scan -sT

我们感兴趣的是了解 TCP 端口是否打开，而不是建立 TCP 连接。因此，一旦通过发送 RST/ACK 确认其状态，连接就会被断开。您可以选择使用 `-sT` 运行 TCP 连接扫描。

如果不是特权用户（root 用户或 sudoer），则 TCP 连接扫描是发现打开的 TCP 端口的唯一可能选项。

`nmap -sT 10.10.247.13`

我们可以用来 `-F` 启用快速模式，并将扫描的端口数量从 1000 个减少到 100 个最常见的端口。

可以添加该 `-r` 选项以连续顺序而不是随机顺序扫描端口。此选项在测试端口是否以一致的方式打开时（例如，当目标启动时）非常有用。

# TCP SYN Scan -sS

默认扫描模式是 SYN 扫描，需要特权（root 或 sudoer）用户。SYN 扫描不需要完成 TCP 3 路握手;相反，一旦收到来自服务器的响应，它就会断开连接。

使该 `-sS` 选项选择此扫描类型。下图显示了 TCP SYN 扫描在未完成 TCP 3 向握手的情况下的工作方式。

`sudo nmap -sS 10.10.95.78`

![](https://cdn.nlark.com/yuque/0/2024/png/39170111/1716556532716-29d23f07-43f7-4473-8012-c95907f83914.png)

**\-sT 与 -sS 比较：**

在下图的上半部分，我们可以看到 TCP 连接扫描 -sT 流量。任何打开的 TCP 端口都需要 Nmap 在关闭连接之前完成 TCP 3 向握手。

在下图的下半部分，我们看到 SYN 扫描 -sS 如何不需要完成 TCP 3 向握手;相反，Nmap 会在收到 SYN/ACK 数据包后发送 RST 数据包。

# UDP Scan -sU

UDP 是一种无连接协议，因此它不需要任何握手即可建立连接。我们无法保证在 UDP 端口上侦听的服务会响应我们的数据包。

但是，如果将 UDP 数据包发送到关闭的端口，则返回 ICMP 端口无法访问错误（类型 3，代码 3）。您可以使用以下 `-sU` 选项选择 UDP 扫描;此外，您可以将其与另一个TCP扫描结合使用。

换句话说，不生成任何响应的 UDP 端口是 Nmap 将声明为打开的端口。

`sudo nmap -sU 10.10.95.78`

# Fine-Tuning Scope and Performance  
微调范围和性能

`-p22,80,443` 将扫描端口 22、80 和 443

`-p1-1023` 将扫描 1 到 1023（含）之间的所有端口

`-p-` 请求扫描所有端口，这将扫描所有 65535 个端口

`-F` 扫描最常见的 100 个端口

`--top-ports 10` 检查十个最常见的端口

`-T<0-5>` 来控制扫描时序。 -T0 是最慢的（偏执狂），而 -T5 最快的是最快的。

> 根据 Nmap 手册页，有 6 个模板：
> 
> -   paranoid (0) 一次扫描一个端口，并在发送每个探测器之间等待 5 分钟
> -   sneaky (1) 在隐身更重要的实际交战中经常使用
> -   polite (2)
> -   normal (3) 默认
> -   aggressive (4)
> -   insane (5) 数据包丢失的可能性增加

`--min-rate=<number>`​`--max-rate=<number>` 控制数据包速率

`--min-parallelism=<numprobes>``--max-parallelism=<numprobes>` 来控制探测并行
