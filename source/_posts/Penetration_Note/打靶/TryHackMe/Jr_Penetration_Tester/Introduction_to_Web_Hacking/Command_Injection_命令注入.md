# Exploiting Command Injection 利用命令注入

使用用户输入用数据填充系统命令的应用程序通常可以合并为意外行为。例如，**shell 运算符** `**;**` **，** `**&**` **和** `**&&**` **组合两个（或多个）系统命令并同时执行它们。**

​  

Linux

| **Payload** | **Description** |
| --- | --- |
| whoami | 查看应用程序在哪个用户下运行。 |
| ls | 列出当前目录的内容。您可以找到配置文件、环境文件（令牌和应用程序密钥）等文件，以及更多有价值的东西。 |
| ping | 此命令将调用应用程序挂起。这在测试盲命令注入应用程序时非常有用。 |
| sleep | 这是测试计算机未 ping 安装盲命令注入应用程序的另一个有用有效负载。 |
| nc | Netcat 可用于在易受攻击的应用程序上生成反向 shell。您可以使用此立足点在目标计算机中导航，以获取其他服务、文件或提升权限的潜在方法。 |

Windows

| **Payload** | **Description** |
| --- | --- |
| whoami | 查看应用程序在哪个用户下运行。 |
| dir | 列出当前目录的内容。您可以找到配置文件、环境文件（令牌和应用程序密钥）等文件，以及更多有价值的东西。 |
| ping | 此命令将调用应用程序挂起。这在测试盲命令注入应用程序时非常有用。 |
| timeout | 此命令还将调用应用程序挂起。如果未安装该 ping 命令，它对于测试盲命令注入应用程序也很有用。 |

# Remediating Command Injection 修正命令注入

**Vulnerable Functions 易受攻击的功能**

在PHP中，许多函数与操作系统交互，通过shell执行命令;这些包括：

-   Exec
-   Passthru
-   System

  

**Input sanitisation 输入消毒**

清理应用程序使用的用户的任何输入是防止命令注入的好方法。这是指定用户可以提交的数据格式或类型的过程。例如，仅接受数值数据或删除任何特殊字符（如 > 和 &/ ）的输入字段。

​  

**Bypassing Filters 绕过过滤器**

应用程序将采用多种技术来过滤和清理从用户输入中获取的数据。这些过滤器会将您限制为特定的有效负载;但是，我们可以滥用应用程序背后的逻辑来绕过这些过滤器。例如，应用程序可能会去掉引号;我们可以改用它的十六进制值来获得相同的结果。
