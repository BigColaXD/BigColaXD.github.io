# 数据库介绍

## mysql

```sql
show databases; #查看当前有哪些数据库
use 数据库名; #进入指定的数据库
show tables; #查询当前数据库下有哪些数据表
select version(),current_date; #显示当前mysql版本和当前日期
select * from 表名 #查询指定表的所有数据
desc user #查询指定表的结构
注意一个点：
MySQL5.0以上版本存在information_schema的数据库，它存储着数据库的所有信息，其中保存着关于MySQL服务器所维护的所有其他数据库的信息。如数据库名，数据库的表，表栏的数据类型和访问权限等。5.0以下没有。
use information_schema;

Mysql有四个系统库：information_schema, performance_schema, sys, mysql
# [information_schema]
保存了mysql服务器所有数据库的信息。比如数据库的库名、数据库的表、数据库表的数据类型，索引，外键、存储过程、视图，函数，事件，触发器等结构信息。
# [performance_schema]
主要用于收集数据库服务器性能参数，可用于监控服务器在一个较低级别的运行过程中的资源消耗、资源等待等情况
# [sys]
库中所有的数据源来自performance_schema，目标是降低performance_schema的复杂度。
# [mysql]
mysql的核心数据库，主要负责存储数据库的用户、权限设置、关键字等mysql自身需要使用的控制和管理信息。
```

```sql
# 验证方式
## [通过 Windows 身份验证进行连接]
使用操作系统中的Windows主体验证帐户名和密码。用户身份由Windows进行确认。 SQLServer不要求提供密码,也不执行身份验证。Windows 身份验证是默认身份验证模式.
## [通过 SQL Server 身份验证进行连接]
使用SQL Server身份验证时，在SQL Server中创建的登录名不基于Windows用户帐户。用户名和密码均通过使用 SQL Server 创建并存储在SQL Server中。
```
